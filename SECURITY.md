# Security Policy

## Supported Versions

This project is an educational assignment and is currently in active development.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it by:

1. **Email**: Contact Tom Burchell at w0516036@nscc.ca
2. **GitHub Issues**: Open a confidential security advisory (if enabled) or a private issue

### What to Include

Please include the following information in your report:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if you have one)

### Response Time

As this is an educational project, we will aim to respond to security reports within:
- **Initial Response**: 48-72 hours
- **Fix Timeline**: Depending on severity, within 1-2 weeks

### Disclosure Policy

- Please do not publicly disclose the vulnerability until a fix has been released
- We will credit you for the discovery (unless you prefer to remain anonymous)

## Security Best Practices

When contributing to this project:
- Do not commit sensitive information (passwords, API keys, etc.)
- Follow secure coding practices
- Keep dependencies up to date
- Run code reviews before merging

## Scope

This security policy applies to:
- All code in the `main` branch
- All released versions
- Documentation that may contain security-relevant information

Thank you for helping keep this project secure!
